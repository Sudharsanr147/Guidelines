Step-by-Step Guide

Firebase Setup & Public Hosting

Complete guide to connect Firebase, set up authentication, Firestore database, and deploy Project Studio Pro to a public domain.

1
Create Your Firebase Project
Takes about 5 minutes. Free forever on Spark plan.
1
Go to Firebase Console
Open console.firebase.google.com and sign in with your Google account.
2
Create a New Project
Click "Add project" → Enter name e.g. project-studio-pro → Disable Google Analytics (optional) → Click Create project.
3
Add a Web App
In your project dashboard, click the </> (Web) icon → Register app with any nickname → Copy the firebaseConfig object shown.
4
Paste Config into index.html
Find the firebaseConfig object in your index.html and replace the placeholder values with your actual config:
Your firebaseConfig (replace in index.html)
const firebaseConfig = {
  apiKey: "AIzaSyABCDEF1234567890abcdef",
  authDomain: "project-studio-pro.firebaseapp.com",
  projectId: "project-studio-pro",
  storageBucket: "project-studio-pro.appspot.com",
  messagingSenderId: "123456789012",
  appId: "1:123456789012:web:abcdef1234567890"
};
2
Enable Authentication
Allow users to sign in with Google or Email/Password
1
Open Authentication in Firebase Console
Left sidebar → Build → Authentication → Click "Get started"
2
Enable Email/Password
Sign-in providers tab → Click Email/Password → Toggle Enable → Save.
3
Enable Google Sign-In
Sign-in providers tab → Click Google → Toggle Enable → Add your support email → Save.
4
Add Authorized Domain (for production)
Authentication → Settings tab → Authorized domains → Add your custom domain (e.g. projectstudiopro.com) after hosting.
3
Set Up Firestore Database
Your data structure and security rules
1
Create Firestore Database
Left sidebar → Firestore Database → Create database → Choose Production mode → Select region closest to your users (e.g. asia-south1 for India) → Enable.
2
Set Security Rules
Firestore → Rules tab → Replace the default rules with the rules below → Publish.
Firestore Security Rules (copy-paste into Rules tab)
// Each user can only read/write their own data
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId;
    }
  }
}
ℹ️
These rules ensure that each user can only access their own data. User A cannot see or modify User B's projects or notes. This is exactly what you need for a multi-user SaaS app.
📂 Your Firestore Data Structure
users/ (collection)
  {userId}/ (document — one per user)
    projects/ (sub-collection)
      {projectId}: { name, branches[], createdAt }
    notes/ (sub-collection)
      {noteId}: { projectId, project, branch, text, date, createdAt }
4
Deploy to Firebase Hosting Free
Your app live at yourproject.web.app in minutes
1
Install Node.js
Download from nodejs.org and install (LTS version). This gives you the npm command.
2
Install Firebase CLI
Open your terminal / command prompt and run:
npm install -g firebase-tools
3
Login to Firebase
firebase login
# A browser window opens — sign in with your Google account
4
Set Up Your Project Folder
Create a folder called project-studio and put your index.html, manifest.json, and sw.js inside a public sub-folder.
# Your folder structure should look like this:
project-studio/
  public/
    index.html    # your main app file
    manifest.json
    sw.js
  firebase.json  # created by firebase init (next step)
5
Initialize Firebase Hosting
Inside your project-studio folder, run:
firebase init hosting

# Answer the questions like this:
# ? Use an existing project → Select your project
# ? Public directory → public
# ? Single-page app? → No
# ? Overwrite index.html? → No
6
Deploy!
firebase deploy

# You'll see something like:
✔ Deploy complete!
Project Console: https://console.firebase.google.com/...
Hosting URL: https://project-studio-pro.web.app
✅
Your app is now live at https://your-project-id.web.app — this is a real public URL anyone can open. Free SSL (HTTPS) is included automatically.
5
Add a Custom Domain
Replace .web.app with your own brand domain
Free (Firebase)
yourapp.web.app
Works immediately after deploy. Good for testing and early launch.
Budget Domain (~₹800/yr)
yourapp.in / .site
Buy from GoDaddy or Hostinger. Connect to Firebase Hosting for free.
Pro Domain (~₹1,200/yr)
yourapp.com
Most trusted for SaaS. Buy from Namecheap, GoDaddy, or Google Domains.
🔗 How to Connect Your Domain to Firebase
1
Open Firebase Hosting → Add custom domain
Firebase Console → Hosting → Add custom domain → Enter your domain (e.g. projectstudiopro.com)
2
Verify ownership
Firebase gives you a TXT record. Add it to your domain registrar's DNS settings (GoDaddy / Namecheap dashboard).
3
Add A records
Firebase provides two IP addresses. Add them as A records in your DNS. Takes 10 minutes to 48 hours to propagate.
4
Done — HTTPS is automatic
Firebase auto-provisions an SSL certificate. Your domain will work with https:// for free.
